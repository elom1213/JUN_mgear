# 컴포넌트 연결(connection) — arm ↔ shoulder

## 문제 요약

`arm_2jnt_01`과 `shoulder_01`을 각각 추가만 하면, 빌드 후 두 컨트롤러가 **서로 무관하게
따로** 생성된다. 어깨를 움직여도 팔이 따라오지 않는다.

원인은 mGear에서 컴포넌트끼리 부모자식/컨스트레인트로 묶어주는 **"컴포넌트 연결
(component connection)"** 설정을 하지 않았기 때문. mGear는 컴포넌트를 그냥 나열한다고
연결해주지 않는다 — **명시적으로 연결을 지정**해야 한다.

## 핵심 개념 — 빌드 후 수동 parent가 아니다

> 빌드된 컨트롤러를 나중에 손으로 parent/constraint 거는 게 아니라,
> **가이드(Guide) 단계에서 컴포넌트를 연결해두고 → 빌드**하면 연결이 자동 생성된다.

즉 순서는 항상 **(가이드에서 연결) → (Rig Builder 실행)**.

## 연결 2단계

### (1) 가이드 re-parent — 계층 연결
`arm_2jnt_01` 가이드의 **루트(root)** 를 `shoulder_01`의 **출력(끝) 조인트** 밑으로 부모를 옮긴다.

- Outliner에서 그냥 드래그하지 말고, mGear의 **Guide Operations re-parent** 기능을 사용하는 걸 권장.
  → 메뉴/절차는 [[02_Shifter_가이드_워크플로우]]의 "Guide Operations(재부모)" 참고.
- 이렇게 하면 팔 컴포넌트가 어깨 컴포넌트의 출력을 부모로 인식한다.

### (2) Connection Type / Connector — 연결 방식 지정
팔 컴포넌트의 **Settings**를 열고 **"Connection Type(Connector)"** 드롭다운에서 연결 방식을 고른다.

- arm 계열 컴포넌트는 **shoulder 연결을 전제로 한 커넥터**가 들어있다. 이걸 골라야
  `shoulder_01`의 출력에 깔끔하게 물린다.
- 연결 방식(standard / parent / orientation 등)에 따라 부모자식이냐 컨스트레인트냐,
  회전만 따라가느냐 등이 달라진다.

이 둘을 마치고 **다시 빌드**하면, 팔 컨트롤러가 어깨 컨트롤러(출력 조인트) 아래로 들어가
부모자식/컨스트레인트가 자동 생성된다.
