## 2026-06-16 — 최초 질문

> **원문 질문**
> "팔이나 다리같은 IK가 필요한 부위에 mgear를 사용해서 IK 컨트롤러 세팅을 하려고 해.
> 이때 아바타 몸의 모든 부위가 아니라 팔, 혹은 다리 부위만 컨트롤 리깅을 하려고 해.
> 팔이 여러 개 달린 크리쳐 아바타 등에 필요해서 몸의 부위만 컨트롤 리깅을 설치하는
> 세팅을 찾고 있어. (mgear4 공식 문서의) 어떤 페이지를 봐야 참고가 되는지 알려줘."

**핵심 의도**
- 전신 리그가 아니라 **부위(팔/다리)만** 골라 IK 컨트롤 리깅.
- **팔이 여러 개 달린 크리쳐** 같은 비표준 구조가 실제 타깃.
- 공식 문서 중 **봐야 할 페이지**를 찾는 것.

**이 질문에서 나온 노트**
- [[01_mgear와_Shifter_개요]] — Shifter가 컴포넌트 조립식이라 부위별 빌드가 가능하다는 답
- [[02_Shifter_가이드_워크플로우]] — 봐야 할 문서 섹션(Guide Manager / Guide Operations / Rig Builder)
- [[03_팔다리_IK_컴포넌트_레퍼런스]] — arm_2jnt_01 / leg_2jnt_01 / control_01
- [[04_다중팔_크리쳐_세팅_노트]] — 다중 팔 크리쳐 실전 흐름
- [[99_참고링크]] — 조사에 사용한 공식 문서 URL

**요약 답**
- 전신 biped 템플릿은 필수가 아니다. Shifter는 빈 가이드에 **필요한 컴포넌트만** 붙여
  빌드할 수 있다 → 부위별·다중 팔 리깅에 적합.
- 우선 볼 페이지: **Shifter User Documentation의 Guide Manager / Guide Operations**.

## 2026-06-16 — 후속 질문 (컴포넌트 연결)

> **원문 질문**
> "`arm_2jnt_01`을 통해 팔 컨트롤러를, `shoulder_01`을 통해 어깨 컨트롤러를 세팅했다.
> 어깨와 팔 컨트롤러가 서로 부모자식 혹은 컨스트레인트로 연관되어 연결되기를 원하는데,
> 지금은 두 컨트롤러가 각각 따로 생성된다. 두 컨트롤러를 연결하려면 공식 문서의 어디를
> 참고하면 되나?"

**핵심 의도**
- 따로 생기는 arm/shoulder 컨트롤러를 **서로 연결**(부모자식/컨스트레인트)하고 싶다.
- 공식 문서 중 **봐야 할 페이지**를 찾는 것.

**요약 답**
- mGear는 컴포넌트를 나열만 한다고 연결해주지 않는다. **가이드 단계에서 연결 → 빌드** 순서.
- (1) `arm_2jnt_01` 가이드 root를 `shoulder_01` 출력 조인트 밑으로 **re-parent**,
  (2) 팔 컴포넌트 Settings의 **Connection Type(Connector)** 지정 후 재빌드.
- 볼 페이지: **Shifter User Documentation(Guide Operations + Connection)** + Component Reference.

**이 질문에서 나온 노트**
- [[05_컴포넌트_연결_노트]] — re-parent + Connection Type 2단계 정리

## 2026-06-16 — 후속 질문 (원본 연결 / 스키닝)

> **원문 질문**
> "어깨·팔 등 몸의 부위 컨트롤러를 의도대로 만들었다. 이제 원본 메시를 움직이는 조인트를
> 컨트롤러와 연결시키고 싶다. 컨트롤러가 움직이면 원본 메시·원본 조인트도 움직이게 하려면
> 어떻게 하나? (먼저 문서 제작용 기획서를 쓰고, 이후 본 노트를 작성)"

**핵심 의도 / 확정 사항**
- 이미 **원본 스켈레톤이 있고 메시가 거기 스킨된 상태** → 원본 조인트를 보존한 채 연결.
- 따라서 **시나리오 B(기존 스켈레톤 연결)** 가 메인.

**요약 답**
- mGear 리그는 자체 디폼 조인트를 만들고, 그건 이미 컨트롤러에 묶여 있다.
  → 원본 조인트가 **mGear 디폼 조인트(`rig_deformers_grp`)를 따라가게** 잇는다.
- B1(권장): 컴포넌트 **Joint Settings에서 기존 조인트에 연결**(이름 + point/orient/scale).
  B2(폴백): 원본 조인트를 mGear 디폼 조인트에 **parentConstraint(+scale)**.
- 원본 메시 스킨은 그대로 유지. 스킨 IO는 `.gSkin/.jSkin/.jSkinPack`.

**이 질문에서 나온 산출물**
- `기획_원본_연결과_스키닝` — 본 노트 제작 기획서
- [[06_원본_연결과_스키닝_노트]] — 실제 설명 노트

## 2026-06-17 — 후속 질문 (모션캡쳐 베이크)

> **원문 질문**
> "Guide Template Manager로 인간형 컨트롤러(EPIC_mannequin_y_up, biped 등)를 만들었다.
> Shogun, OptiTrack Motive 같은 광학식 모캡 장비로 얻은 데이터(본에 키프레임만 있음)를
> 이 컨트롤러에 베이크하고 싶다. 그 방법을 문서로 정리하고 싶다."

**핵심 의도 / 확정 사항**
- 입력: **본에 키프레임만** 박힌 모캡 스켈레톤(Shogun/Motive, FBX/BVH).
- 목표: 그 모션을 mGear **컨트롤러**에 베이크해 애니메이터가 편집 가능하게.
- 문서 구조: 기획서 없이 **본 노트 한 개만** 작성. 메인=HumanIK, 기준=mGear 5.x.

**요약 답**
- 모캡 본과 mGear 컨트롤 리그는 스켈레톤·비율·네이밍이 달라 **리타게팅**이 필요.
- 메인(A): **HumanIK**로 모캡(Source)→mGear 리그(Target) 리타게팅 후,
  mGear **HumanIK Mapper**(Mocap 서브메뉴, Sub IK)로 **컨트롤러에 Bake**.
- 폴백(B): 컨트롤을 모캡 본에 **컨스트레인트 → Bake Simulation → 컨스트레인트 삭제**.
- 준비: 업축(y_up)·스케일(cm)·싱글 루트·T-pose·fps 정합. 베이크 후 Euler 필터.
